发布页面重构设计方案

日期：2026-06-07

背景​

当前 /publish 页面存在以下问题：

  • 编辑体验差：Markdown 编辑器和代码块分离，不直观
  • 流程不顺畅：每次都要手动输入 GitHub Token，没有草稿保存，缺少确认机制
  • 功能不够用：缺少图片上传、文章管理（编辑/删除已有文章）等功能

目标​

打造一个完整的文章发布管理系统，提升发布体验。

架构方案​

整体布局：单页 Tab 切换​

密码验证后进入主页面，通过顶部 Tab 切换三个视图：

┌─────────────────────────────────────────┐
│  🔐 密码验证通过后                        │
│  ┌─────────┬─────────┬──────────┐       │
│  │  写文章  │  草稿箱  │ 文章管理  │       │
│  └─────────┴─────────┴──────────┘       │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │
│  │       对应 Tab 的内容区域        │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘

数据流​

数据类型存储位置说明
草稿localStorage键名 drafts，每条含 id/title/content/category/tags/description/codeBlocks/updatedAt
已发布文章GitHub API读取仓库 docs/ 目录下的 .md 文件
图片GitHub API上传到仓库 static/img/ 目录，返回可访问 URL
发布/编辑/删除GitHub APIContents API 的 GET/PUT/DELETE

认证方式​

保持现有密码验证机制，进入后 Token 在发布时输入并临时使用。

Tab 1：写文章​

布局​

┌─────────────────────────────────────────┐
│ 标题: [________________________]        │
│ 分类: [前端 ▾]   标签: [React, Hooks]   │
│ 摘要: [简短描述...]                     │
├─────────────────────────────────────────┤
│ [B] [I] [H1] [H2] [🔗] [📷] [💻] [👁] │  ← 工具栏
├─────────────────────────────────────────┤
│                                         │
│         Markdown 编辑区域               │
│     (支持拖拽/粘贴图片自动上传)          │
│                                         │
├─────────────────────────────────────────┤
│ [保存草稿]              [预览] [发布]   │
└─────────────────────────────────────────┘

功能说明​

  • 元信息表单：标题（必填）、分类（下拉选择）、标签（逗号分隔）、摘要
  • 工具栏：
    • B I：加粗/斜体
    • H1 H2：标题级别
    • 🔗：插入链接
    • 📷：触发图片上传
    • 💻：插入代码块模板
    • 👁：切换全屏预览
  • 图片上传：拖拽/粘贴图片自动调用 GitHub API 上传到 static/img/，返回 URL 插入 Markdown
  • 保存草稿：存入 localStorage，支持自动保存（编辑 30 秒无操作自动保存）
  • 预览：弹出全屏预览弹窗，渲染最终效果
  • 发布：弹出确认弹窗，输入 GitHub Token 后调用 API 发布

编辑模式​

当从草稿箱或文章管理加载内容时，编辑器进入编辑模式：

  • 顶部显示提示条："正在编辑：文章标题 [取消编辑]"
  • 发布时更新已有文件而非创建新文件

布局​

│                                         │

功能说明​

  • 显示所有 localStorage 中的草稿，按 updatedAt 倒序
  • 支持标题搜索过滤
  • 「编辑」→ 切换到写文章 Tab 并加载草稿内容
  • 「删除」→ 二次确认后删除
  • 「清空所有草稿」→ 二次确认后清空

Tab 3：文章管理​

布局​

┌─────────────────────────────────────────┐
│ 已发布文章 (12)      [搜索文章...] 🔍   │
├─────────────────────────────────────────┤
│ 分类筛选: [全部 ▾]  排序: [最新发布 ▾]  │
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │ 📄 React Hooks 入门指南              │ │
│ │ 前端开发 | 2026-06-01 发布           │ │
│ │ docs/frontend/react-hooks.md        │ │
│ │              [编辑] [删除] [查看]    │ │
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ 📄 二叉树遍历算法                    │ │
│ │ 算法学习 | 2026-05-28 发布           │ │
│ │ docs/algorithm/binary-tree.md       │ │
│ │              [编辑] [删除] [查看]    │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘

功能说明​

  • 通过 GitHub API 递归读取 docs/ 下所有 .md 文件
  • 解析每个文件的 frontmatter（title、date、tags、description）
  • 支持按分类筛选、按日期排序、标题搜索
  • 「编辑」→ 获取文件内容（含 SHA），加载到写文章 Tab，进入编辑模式
  • 「删除」→ 二次确认后通过 GitHub API 删除文件（需要 SHA）
  • 「查看」→ 在新标签页打开已发布的文章 URL

技术要点​

GitHub API 集成​

// 获取文件列表
GET /repos/{owner}/{repo}/contents/{path}

// 获取文件内容（含 SHA，编辑/删除需要）
GET /repos/{owner}/{repo}/contents/{path}

// 创建/更新文件
PUT /repos/{owner}/{repo}/contents/{path}
Body: { message, content (base64), sha (更新时需要) }

// 删除文件
DELETE /repos/{owner}/{repo}/contents/{path}
Body: { message, sha }

图片上传流程​

  1. 用户拖拽/粘贴图片或点击 📷 按钮
  2. 读取文件为 base64
  3. 调用 GitHub API PUT 到 static/img/{filename}
  4. 返回 URL（/img/{filename}）
  5. 在编辑器光标位置插入 ![描述](url)

草稿自动保存​

  • 编辑内容变化时启动 30 秒防抖定时器
  • 定时器到期自动保存到 localStorage
  • 手动保存按钮立即保存
  • 保存时显示轻量提示："草稿已保存"

动画​

  • Tab 切换：内容区域淡入过渡
  • 列表项：GSAP stagger 入场动画
  • 弹窗：GSAP scale + fade 动画
  • 保持现有 gsap.set + gsap.to 模式

深色模式​

所有新增组件使用 CSS Modules，通过 [data-theme='dark'] 选择器适配深色主题。
