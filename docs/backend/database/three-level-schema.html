三层模式与二级映射

一、三层模式​

1.内模式​

  • 作用 ：描述数据在物理存储介质上的实际存储方式。

  • 内容 ：比如数据是否压缩、如何建立索引、数据块的大小、存储路径等。

  • 视角 ：这是数据库系统 （如DBMS）和操作系统 看到的数据视图。

2.概念模式​

  • 作用 ：描述数据库中所有数据的逻辑结构 ，即"数据是什么，以及它们之间有什么联系"。

  • 内容 ：比如一个"学生表"包含学号、姓名、班级等字段，以及这些表之间的关联关系。它不关心数据具体存在哪个硬盘扇区。

  • 视角 ：这是数据库管理员 (DBA) 和** 应用设计者**看到的数据视图。整个数据库只有一个概念模式。

3.外模式​

  • 作用 ：也叫用户模式 或子模式。它是针对某个特定用户或应用，所定义的概念模式的一个局部视图 。

  • 内容 ：用户只能看到和操作他需要的数据，而无需关心其他无关数据。例如，教务处系统能看到所有学生信息，而学生本人只能查询自己的信息。不同的用户可以有完全不同的外模式。

  • 视角 ：这是最终用户 或应用程序 看到的数据视图。可以有多个外模式。

二、二级映射​

1.外模式/概念模式映射​

  • 作用 ：将用户看到的外模式（局部逻辑）映射到全局的概念模式。

  • 效果 ：当概念模式改变 时（例如：修改了数据库的全局逻辑结构），只要修改这个映射（通常是重新定义外模式），就可以让外模式保持不变 。这样，基于该外模式编写的应用程序就不需要修改 。

  • 这就是"逻辑数据独立性"。

2.概念模式/内模式映射​

  • 作用 ：将全局的逻辑结构（概念模式）映射到物理存储结构（内模式）。

  • 效果 ：当内模式改变 时（例如：更换了性能更好的硬盘，或为了优化查询而重建了索引），只要修改这个映射，就可以让概念模式保持不变 。由于外模式依赖概念模式，所以上层应用也都不受影响。

  • 这就是"物理数据独立性"。

三、优点​

  1. 高度的数据独立性： 有效分离应用程序与数据的逻辑、物理结构，降低系统耦合度，提升系统的可维护性。

  2. 提升数据共享能力： 不同的用户可以基于各自的外模式访问同一份数据，实现多角度、多需求的数据共享，提高数据利用率。

  3. 增强数据安全性与保密性： 通过外模式可以限制用户访问的数据范围，结合权限管理机制可以有效保障数据安全。

  4. 简化应用开发： 开发者无需关注数据的存储细节，只需面向外模式或模式开发，降低开发难度，提升开发效率。

总结​

概括： 三层模式（外、概念、内）从三个不同角度描绘数据，将用户逻辑、全局逻辑和物理存储彻底分开；二级映射则在它们之间搭建桥梁，使得修改内层或中间层时，不影响上层的应用，从而实现了数据独立性。
