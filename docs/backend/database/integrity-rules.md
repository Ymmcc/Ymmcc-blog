## 一、基本概念

### 1.实体完整性

-   **定义**：主键的值不能为空且必须唯一。
-   **解释**：每个记录都必须有一个唯一标识，确保数据表中的每一行都是可区分的。
-   **例子**：在"学生表"中，学号被设为主键，则每个学生的学号必须唯一，且不能为空。

### 2.参照完整性

-   **定义**：外键的值必须等于被参照关系中某个元组的主键值，或者为空值。
-   **解释**：表与表之间的关系通过外键维护，外键的值必须在被引用的表中存在。
-   **例子**："选课表"中的学号必须存在于"学生表"中，否则就是无效数据。

### 3.用户定义完整性

-   **定义**：针对某一具体应用领域的约束条件。
-   **解释**：由用户根据实际业务需求定义的规则，允许范围内的值。
-   **例子**：年龄必须在 0~150 之间，性别只能是"男"或"女"，成绩在 0~100 之间。

## 二、具体实例

-   **实体完整性**：向学生表插入一条新记录时，学号不能为空，也不能与已存在的学号重复。
-   **参照完整性**：在选课表中插入选课记录时，学号必须在学生表中存在，课程号必须在课程表中存在。
-   **用户定义完整性**：学生表的年龄字段，插入时检查是否在合理范围内（0~150 岁）。
